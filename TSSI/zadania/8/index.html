:active - stylizuje element, gdy jest jest aktywny, czyli od momentu naciśnięcia przycisku myszy do jego zwolnienia. Gdy aktywujesz poniższy przycisk, zmieni on kolor na czerwony.

:link - stylizuje element, jeśli jest on nieodwiedzonym linkiem. Poniższy link wyświetlany jest w kolorze czerwonym - odwiedzenie go sprawi, że nie będzie już w taki sposób formatowany. Przycisk REFRESH pozwoli ci na otrzymanie nowego, nieodwiedzonego linku.

link
:visited - stylizuje element, jeśli jest on odwiedzonym już wcześniej linkiem. Poniższy link wyświetlany jest w domyślnym, niebieskim kolorze - odwiedzenie go sprawi, że zmieni kolor na czerwony. Przycisk REFRESH pozwoli ci na otrzymanie nowego, nieodwiedzonego linku.

:checked - stylizuje element (radio, checkbox), jeśli jest on zaznaczony.

:default - stylizuje element (radio, checkbox, option), który po załadowaniu strony jest wartością domyślną.

:disabled / :enabled - stylizuje element (generalnie input), jeśli jest on zablokowany (disabled) / odblokowany (enabled).

:focus / :focus-within - stylizuje element (generalnie input), jeśli jest na nim focus / jeśli dowolny child ma focus.

:indeterminate - stylizuje element (radio, checkbox), jeśli jest wartość nie jest określona (żaden radio nie jest zaznaczony; tylko JS można dodać atrybut indeterminate).

:in-range / :out-of-range - stylizuje element (input gdzie mamy min/max), jeśli wartość mieści sie w przedziale / jest poza przedziałem.

:invalid / :valid - stylizuje element (input z patternem, w tym URL i EMAIL + form), jeśli wartość nie zgadza się z patternem / zgadza się z patternem.

:optional / :required:required - stylizuje element (generalnie input), jeśli nie ma atrybutu required / jeśli ma on atrybut required.

:placeholder - stylizuje element (generalnie input), jeśli placeholder jest pokazany (brak wartości np. w text).

:read-write / :read-only - stylizuje element (generalnie input), jeśli jest on read-write (bez readonly czy disabled) / posiada on atrybut readonly (odziwo mimo, że element posiada atrybut disabled jest uznawany za read-write, więc w3school mnie okłamało 😕).
